Desafio: Abstraindo um Bootcamp com Java e POO
Olá! Eu sou a Alice e este é o meu repositório para o desafio de projeto da DIO (Digital Innovation One). O objetivo deste projeto foi aplicar os pilares da Programação Orientada a Objetos (POO) em um cenário real de gerenciamento de Bootcamps.

🛠️ Evoluções e Diferenciais
Para ir além da proposta básica e praticar a "evolução interessantes" sugerida, implementei:

Níveis de Dificuldade: Adicionei o atributo nivel (Básico, Intermediário e Avançado) na classe Curso.

Cálculo de XP Dinâmico: Utilizei polimorfismo para que o XP seja calculado com base no nível; cursos avançados rendem bônus maior, refletindo a complexidade do aprendizado.

🧱 Pilares de POO Aplicados
O código foi estruturado para demonstrar os fundamentos da orientação a objetos:

Abstração: Uso de classe abstrata Conteudo para representar itens do bootcamp.

Encapsulamento: Proteção dos dados com modificadores private e métodos de acesso getters/setters.

Herança: As classes Curso e Mentoria herdam atributos comuns de Conteudo.

Polimorfismo: Implementações distintas do método calcularXp() para cada tipo de objeto.

💡 Mentoria e Visão de Carreira
Durante o desenvolvimento, utilizei o Google Gemini como um mentor técnico para entender melhor as mensagens de erro do compilador Java e para discutir a melhor forma de implementar a lógica de níveis. Isso me ajudou a fixar os conceitos de forma prática.

Por que aprender o básico e a ler código?
Meu foco de carreira está em Consultoria e Inovação. Acredito que, para inovar com segurança, é indispensável compreender os fundamentos. Saber ler código e dominar a lógica básica me dá a base técnica necessária para desenhar soluções estratégicas, entender as limitações do desenvolvimento e dialogar de forma eficiente com times de tecnologia.
